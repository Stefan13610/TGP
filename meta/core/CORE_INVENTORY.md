# CORE_INVENTORY — mapa rdzenia teorii

> **Cel pliku:** jednolinijkowy opis każdej sekcji `core/sek0Z_*` /
> `axioms/*` / `partial_proofs/*` z linkiem "Promoted from research/...".
> Single source of truth dla pytania "czy X jest już w core".
>
> **Read-mostly dla agentów.** Wpisy dodaje się **tylko** przez akt
> promocji (`CORE_INTAKE.md` workflow), wykonywany przez człowieka.

---

## 0. Reguły wpisu

1. **Jeden wiersz = jeden wynik core**, nie sekcja LaTeX. Jeśli sekcja
   `sek08c.tex` zawiera 4 niezależne wyniki, to są 4 wiersze.
2. **`Promoted from`** musi cytować konkretny folder w `research/` lub
   `axioms/` lub powiedzieć `pre-existing` (gdy core powstał przed
   wprowadzeniem workflow).
3. **`Last verified`** — data ostatniego sprawdzenia, że wpis odpowiada
   aktualnej zawartości pliku core. Po edycji core należy zaktualizować
   datę (lub przesunąć do `OUT-OF-DATE`).
4. **Wpis nie jest usuwany** po zmianie core — zostaje z notatką
   `superseded by <ID>` i ścieżką do nowej wersji.

## 3. Czego inventory NIE robi

- **Nie zastępuje `INDEX.md`** — tamten linkuje pliki nawigacyjnie.
  Inventory mapuje **wyniki** → **źródła**.
- **Nie zastępuje `DEPENDENCIES.md`** — tamten jest auto-generowanym
  grafem `\input`/`\ref`/`[[...]]` zależności LaTeX. Inventory jest
  ludzkojęzyczny.
- **Nie wnioskuje** — nie próbuje samodzielnie ustalić, co należy do
  core przez analizę treści `main.tex`. Jedyne źródło prawdy to akt
  promocji (`INTAKE_*.md` zaakceptowany przez człowieka).
